assessment c back to ToC or Class ToC

IRI: https://w3id.org/con-tax/properties/cp#Assessment

An instance of cp:Assessment represents an assessment related to a certain construction-related object (building, bridge, space, component, etc.), connected with the cp:hasAssessment property. Starting from an assessment node, one assessment score (cp:assessmentScore or subproperties) relation can be connected to link to the scoring value.

has assessmentop back to ToC or Object Property ToC

IRI: https://w3id.org/con-tax/properties/cp#hasAssessment

A property to connect an assessed object (construction component, spatial zone or damage) to an assessment node. The assessment node can be connected to (1) its assessment score using (a subproperty of) cp:assessmentScore and (2) the inspection context of the said assessment using cp:assessmentFromInspection. Property inspired by Monumentenwacht reports and the DOT extension DASB.

  • assessment score

assessment scoredp back to ToC or Data Property ToC

IRI: https://w3id.org/con-tax/properties/cp#assessmentScore

Property to connect a node representing an assessment node (cp:Assessment) with incoming cp:hasAssessment relation from an assessed object, towards its assessment score (literal). This property is made functional, meaning that each cp:Assessment instance node can only have one outgoing cp:assessmentScore. More specific subproperties have to be defined in ontology extensions to specify the assessed aspect and the applied scoring system. Property inspired by Monumentenwacht reports and the DOT extension DASB.
Is defined by
https://w3id.org/con-tax/properties/cp
